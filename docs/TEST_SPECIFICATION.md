# 測試規格

## 1. 登入流程

### 1.1 測試目標
驗證帳號能透過遊戲 URL 成功登入，並完成遊戲初始化。

### 1.2 測試步驟
1. 初始化 WebSocket 監聽
2. 導航至遊戲 URL
3. 等待 Canvas 載入（最多 30 秒）
4. 等待 Cocos Creator 場景載入（最多 2.5 秒）
> PWA 關閉會在「進入房間流程」中、讀到房間列表後執行

### 1.3 登入規則

| 規則項 | 說明 | 來源 |
|--------|------|------|
| WebSocket 監聽 | 必須在登入前初始化，用於監聽遊戲事件 | 技術文檔 |
| Canvas 載入超時 | 最多等待 30 秒，超時則測試失敗 | 技術文檔 |
| 場景載入檢查 | 檢查 `cc.director.getScene()` 和 Canvas 節點是否存在 | 技術文檔 |
| PWA 檢測方式 | 檢查 `Layer_Loading` 是否有子節點<br>有子節點 = 有 PWA<br>沒有子節點 = 沒有 PWA | 技術文檔 |
| PWA 關閉路徑 | `Layer_Loading/PWAPopupView/TouchImage` | 技術文檔 |
| PWA 關閉方法 | 觸摸 `TouchImage` 節點（發送 `TOUCH_START` 和 `TOUCH_END` 事件） | 技術文檔 |
| PWA 關閉確認 | 觸摸後必須再次檢查 `Layer_Loading` 是否仍有子節點；若仍存在子節點則改用 `PWAPopupView.active = false` 強制關閉 | 技術文檔 |
| PWA 彈窗處理 | 若存在 PWA 安裝提示，必須關閉後才能繼續 | 產品規格 |

### 1.4 驗證點
- Canvas 元素已載入
- `cc.director.getScene()` 可用
- Canvas 節點存在於場景中
- PWA 彈窗已關閉（如存在）

---

## 2. 進入房間流程

### 2.1 測試目標
驗證能成功進入指定房間，並完成房間初始化。

### 2.2 測試步驟
1. 等待房間列表載入
2. 若 `Layer_Loading` 仍有子節點，觸摸 `PWAPopupView/TouchImage` 關閉 PWA；如未消失則設定 `active=false`
3. 進入目標房間（如 CG500X、CGIGOJP1、CGT01）
4. 開啟下注面板（用於顯示路書）
5. 嘗試讀取抽水率（最多輪詢 3 次 × 100ms）

### 2.3 進房規則

| 規則項 | 說明 | 來源 |
|--------|------|------|
| 房間列表載入 | 必須等待房間列表可見後才能進入房間 | 產品規格 |
| PWA 處理 | 房間列表載入後立即關閉 PWA：先觸摸 `TouchImage`，若 `Layer_Loading` 仍有子節點則改用 `PWAPopupView.active = false` | 技術文檔 |
| 下注面板開啟 | 進房後必須開啟下注面板，路書才能正常顯示 | 技術文檔 |
| 抽水率讀取 | 進房後立即嘗試讀取，若失敗則輪詢 3 次 | 技術文檔 |
| 抽水率路徑 | `table.round._data.data.jackpotInfoList[702].deductionRateList[0].deductionRate` | 技術文檔 |

### 2.4 驗證點
- 房間列表已載入
- 成功進入目標房間（`roomResult.success === true`）
- 下注面板已開啟
- 抽水率讀取成功（可選，僅 Jackpot 房間）

---

## 3. 等待開局流程

### 3.1 測試目標
驗證能正確等待新開局，並偵測房間模式。

### 3.2 測試步驟
1. 等待 `openround` 事件（最多 60 秒）
2. 偵測房間模式（500X/Speed/Jackpot）
3. 記錄房間模式資訊

### 3.3 開局規則

| 規則項 | 說明 | 來源 |
|--------|------|------|
| openround 事件 | 必須等待 `openround` 事件後才能下注 | 產品規格 |
| 開局超時 | 最多等待 60 秒，超時則測試失敗 | 技術文檔 |
| 房間模式偵測 | 根據 `subType` 判斷：500X、Speed、Jackpot | 技術文檔 |
| 下注時機 | 開局後立即下注，避免錯過下注時間窗口 | 產品規格 |

### 3.4 驗證點
- 成功獲取新開局號碼（`newRoundNumber !== null`）
- 房間模式偵測成功（`ROOM_MODE.is500x`、`ROOM_MODE.isSpeed`、`ROOM_MODE.isJackpotV2`）

---

## 4. 下注流程

### 4.1 測試目標
驗證能成功下注，並正確扣款。

### 4.2 測試步驟
1. 開局後立即執行下注
2. 記錄開局時路書快照（用於派彩後比對）
3. 讀取開局後 Jackpot 資訊（抽水率、獎池金額等）

### 4.3 下注規則

| 規則項 | 說明 | 來源 |
|--------|------|------|
| 下注時機 | 開局後立即下注，優先下注避免錯過時間窗口 | 產品規格 |
| 錢包扣款 | `walletBefore - walletAfter = totalBetAmount` | 產品規格 |
| 路書快照 | 下注後立即記錄路書，避免讀到上一局 | 技術文檔 |
| Jackpot 資訊 | 開局後讀取：原始獎金、抽水率、預期獎池 | 技術文檔 |
| 抽水計算 | `抽水 = 總下注金額 × 抽水率%` | 產品規格 |
| 預期獎池 | `預期獎池 = 開局前獎池 + 抽水` | 產品規格 |
| roundCode | 必須獲取到 `roundCode`，否則無法進行派彩驗證 | 技術文檔 |

### 4.4 驗證點
- 下注成功（`betResult.success === true`）
- 錢包扣款正確（`walletBefore - walletAfter = totalBetAmount`）
- 下注區域金額正確
- 獲取到 `roundCode`
- 開局時路書已記錄
- Jackpot 資訊已讀取（僅 Jackpot 房間）

---

## 5. 派彩流程

### 5.1 測試目標
驗證派彩正確，包括錢包增加、路書更新、各區域派彩詳情。

### 5.2 測試步驟
1. 等待派彩事件（最多 60 秒）
2. 驗證錢包變化
3. 讀取派彩後路書（輪詢直到與開局快照不同）
4. 驗證路書顏色與派彩序列匹配
5. 計算各區域派彩詳情
6. 驗證轉盤/刮刮樂結果（僅 Jackpot 房間）

### 5.3 派彩規則

| 規則項 | 說明 | 來源 |
|--------|------|------|
| 派彩超時 | 最多等待 60 秒，超時則測試失敗 | 技術文檔 |
| 路書更新 | 派彩後路書會更新，需輪詢直到與開局快照不同 | 技術文檔 |
| 路書匹配 | 路書第一個顏色對應區域 = 派彩第一個區域 | 產品規格 |
| 基本賠率（1 hit） | 1:1（不含本金）→ 2x（含本金） | 產品規格 |
| 基本賠率（2 hits） | 1:2（不含本金）→ 3x（含本金） | 產品規格 |
| 基本賠率（3 hits） | 1:3（不含本金）→ 4x（含本金） | 產品規格 |
| Speed 電子骰匹配（1） | 1:1.3（不含本金）→ 2.3x（含本金） | 產品規格 |
| Speed 電子骰匹配（2） | 1:2.5（不含本金）→ 3.5x（含本金） | 產品規格 |
| Speed 電子骰匹配（3） | 1:8（不含本金）→ 9x（含本金） | 產品規格 |
| 500X 電子骰（2同色） | 使用 143 Bonus 中的 `rate` | 產品規格 |
| 500X 電子骰（3同色） | 使用 143 Bonus 中的 `rate` | 產品規格 |
| Any Double (807) | 存在二同且非三同時中獎，使用 143 Bonus 中的 `rate` | 產品規格 |
| Any Triple (808) | 三同時中獎，使用 143 Bonus 中的 `rate`（預設 25x） | 產品規格 |
| Jackpot 轉盤倍率 | 從 `payout d.v['10']['83'][0]` 讀取，-1 表示 JP | 技術文檔 |
| Jackpot 刮刮樂 | 100X 或 JP，從 `openBonus` 事件讀取 | 產品規格 |
| JP 派彩計算 | `JP = 獎池（開局前 + 本局抽水）+ 本金` | 產品規格 |

### 5.4 驗證點
- 派彩事件被偵測到（`payoutResult.detected === true`）
- 錢包增加金額正確
- 路書顏色與派彩序列匹配（第一個顏色對應區域 = 派彩第一個區域）
- 各區域派彩詳情正確（根據 hits 數量計算賠率）
- **500X 房間**：電子骰結果（807/808）、143 Bonus 資訊
- **Speed 房間（CGT01）**：電子骰匹配倍率
- **Jackpot 房間（CGIGOJP1）**：轉盤倍率、刮刮樂結果、抽水率計算、JP 獎池計算

---

## 6. 完整流程整合測試

### 6.1 測試目標
驗證從登入到派彩的完整遊戲流程，包括下注、派彩、路書、抽水率等核心功能。

### 6.2 前置條件
1. 測試帳號已配置（`config/testConfig.js` 中的 `betting` 帳號）
2. 環境變數 `TEST_ACCOUNT=betting`（可選，預設使用 `default`）
3. 目標房間已配置（`config/testConfig.js` 中的 `full-flow.targetRoom`）
4. 下注配置已設定（`config/testConfig.js` 中的 `full-flow.bets`）

### 6.3 測試步驟
1. **登入網址**：初始化 WebSocket 監聽，導航至遊戲 URL
2. **關閉 PWA**：關閉 PWA 安裝提示彈窗
3. **進入指定房間**：進入目標房間，開啟下注面板，嘗試讀取抽水率
4. **等待開局**：等待 `openround` 事件，偵測房間模式
5. **下注**：開局後立即執行下注，記錄開局時路書快照，讀取開局後 Jackpot 資訊
6. **派彩**：等待派彩事件，驗證錢包變化、路書更新、各區域派彩詳情、轉盤/刮刮樂結果

### 6.4 測試配置

| 配置項 | 說明 |
|--------|------|
| 測試超時 | 3 分鐘（180 秒） |
| 視窗大小 | 414×896（模擬手機） |
| 提前結束條件 | 若錢包未扣款（下注時間已過）或未獲取到 `roundCode`，會提前結束測試 |

### 6.5 最終報告
測試完成後會顯示：
1. 下注錢包扣款：通過
2. 下注區域金額：通過
3. 派彩錢包增加：檢測到
4. 路書顏色/倍率：顯示路書顏色和倍率
5. 抽水檢核：顯示抽水率、抽水計算、JP 獎池（如適用）
