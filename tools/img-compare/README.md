# 🖼️ 圖片比對工具

本機啟動的網頁工具：在**同一套擷取規則**下，比對兩個網址（例如本機開發 vs 原始環境）所載入的**圖片資源**，並提供大小、HTTP 成敗、視覺差異與 CSV 匯出。

---

## 使用目的（給 QA／開發）

開發端進行**圖片壓縮、拆分、合併圖檔或刪除圖片**後，QA 需確認上線前後是否符合預期。以往多依賴瀏覽器 **Network** 手動篩選 `img` 請求；本工具改為**一次列出**成功／失敗、張數、合計體積與 A/B 並排比對，**一目瞭然**，減少遺漏與口述對照成本。

---

## 限制與使用建議（請先讀）

| 項目 | 說明 |
|------|------|
| **時間窗快照** | 擷取會在「靜默一段時間無新圖片請求」或「達最長等待」後結束，**無法保證**與 DevTools 在任意時間點的每一筆請求完全一致，尤其頁面有**輪詢、WebSocket、需操作才載入**的資源時。 |
| **擷取模式** | 有**定時刷新列表圖**的頁面請選「**含列表輪詢**」，否則可能提早收斂、少抓到後續一波圖。 |
| **A/B 網址** | 建議兩邊為**相同操作流程**（同 query、同是否已進房／同一畫面），否則差異可能來自**流程不同**而非圖檔變更。 |
| **H5／行動版** | 若目標為手機版頁面，請確認在 Playwright 設定的 viewport 下，實際載入的圖與你手動測試時一致（必要時再調整 `server.js` 內 `viewport`／UA）。 |

以上屬工具**邊界**，不影響「資源清單＋成敗＋體積」作為驗收主流程的價值。

## 安裝步驟（只需做一次）

```bash
cd tools/img-compare
npm install
```

若尚未安裝 Playwright 瀏覽器：

```bash
npx playwright install chromium
```

## 啟動

```bash
npm start
# 或
node server.js
```

於瀏覽器開啟：**http://localhost:3000**

## 使用方式

1. 在「A 版（本機開發）」輸入本機或測試環境網址  
2. 在「B 版（原始環境）」輸入對照環境網址（可留空）  
3. 依頁面行為選擇 **擷取模式**（見下表）  
4. 點擊「🚀 開始擷取」  
5. 等待已填寫的網址完成擷取（含捲動與延遲載入觸發）  
6. 於 **總覽／載入狀態／資源大小／視覺比對** 查看結果，必要時匯出 CSV  

> 單站檢視：只填 A 或只填 B 也可執行，適合快速查看單一網址的圖片載入狀態、失敗請求與大小分布。

### 擷取模式（不需自行填秒數）

| 模式 | 適用情境 | 行為概要 |
|------|----------|----------|
| **標準** | 一般頁面、圖片多為一次性載入 | 較短靜默與總等待；允許「重複圖」達門檻後提前收斂 |
| **含列表輪詢** | 房間列表等**定時刷新圖片** | 較長靜默與總等待、捲動時間略長；須先滿足**最少擷取時間**才允許用「靜默」收斂；**關閉**重複圖提前結束，降低輪詢誤判 |

## HTTP 成功／失敗（與 DevTools 概念對齊）

| 分類 | 條件 |
|------|------|
| **成功（納入比對張數與合計體積）** | `200–299` 且 `Content-Type` 為 `image/*`；或 **`304`**（快取命中，視為有載到；可能無 response body，介面可能無預覽、視覺 diff 可能略過） |
| **失敗（單獨列表）** | `4xx`／`5xx`；或 HTTP 成功但回傳**非圖片**（例如 404 回傳 `application/xml`） |

比對、配對、資源大小、視覺 diff **僅使用「成功」清單**。失敗列於「📡 載入狀態」與總覽計數，可匯出 CSV。

---

## 分頁說明

| 分頁 | 功能 |
|------|------|
| 📊 **總覽** | 成功／失敗筆數、A/B **成功圖片合計位元組**、B 相對 A 體積%、已配對／未配對、大小異常、手動配對入口、匯出 |
| 📡 **載入狀態** | A／B 各自「成功」與「失敗」表格（HTTP、類型、錯誤摘要等）；寬螢幕下 A/B 並排；表區可獨立捲動；成功筆數多時預設可摺疊 |
| 📦 **資源大小** | 逐張 A/B 大小、HTTP、差異百分比、篩選 |
| 🖼️ **視覺比對** | Canvas 像素差異（兩邊皆須有圖片本體；304 無 body 時無法 diff） |

## 配對邏輯

- **自動配對**：先依檔名（正規化後）配對，剩餘再依載入順序  
- **手動配對**：張數不一致時，於總覽點「🔗 手動配對」指定對應關係  

## 縮圖與 Lightbox

- 點縮圖可開啟 Lightbox（A／B／差異圖）  
- 鍵盤左右切換、Esc 關閉  

## 匯出

- 總覽：比對結果 CSV、失敗清單 CSV  
- 載入狀態：失敗清單 CSV  
- 資源大小／視覺比對：各分頁專用 CSV

## 技術說明

- **擷取**：Playwright 開啟頁面，攔截 `resourceType` 為 `image` 的 HTTP 回應（含多數 JS 觸發的圖片）  
- **延遲載入**：自動捲動頁面以觸發 lazy-load  
- **視覺比對**：前端以 Canvas 讀取像素；差異超過門檻之像素於 diff 圖標示  
- **代理**：圖片經本機 `/api/img/...` 提供，避免瀏覽器 CORS 阻擋 Canvas 讀取  
- **效能**：視覺比對分批非同步執行，避免介面長時間卡住  

## 注意事項

- 圖片資料暫存於**記憶體**（session），重啟 `server.js` 後需重新擷取  
- 單張超過 **8 MB** 的成功圖會略過  
- 成功圖最多 **300** 張、失敗記錄最多 **200** 筆（可於 `server.js` 調整 `MAX_OK`／`MAX_FAIL`）  
- 相同「檔名 + 大小 + Content-Type」的成功圖會**去重**，與 DevTools「每一筆請求一行」的計數可能不同
